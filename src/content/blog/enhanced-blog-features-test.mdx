## Inline text and formatting

Body text supports **bold**, *italic*, ~~strikethrough~~, `inline code`, and ==highlighted phrases== that read like a marker pen. Links look like [this one to the homepage](/), and you can drop a footnote at the end of a thought.[^note] Highlights and footnotes are the two most recent additions, so they are worth a second look.

Type a superscript like x² or a fraction inline with math, which the next section covers in full.

## Headings and structure

The floating table of contents on the left is built from your headings, so keep them meaningful. This post uses `##` for sections and `###` for subsections.

### A third-level heading

Subsections nest under their parent in the table of contents and get a slightly smaller size.

#### A fourth-level heading

Fourth level is the deepest the design styles. Anything past it reads as body text, so stop here.

## Lists

Unordered lists hold loosely related points:

- A first point, long enough to wrap onto a second line so you can see the hanging indent
- A second point
- A third point with **bold** and a bit of `code`

Ordered lists carry steps in sequence:

1. Clone the repository
2. Install dependencies
3. Run the dev server and open the preview

Nest a list to group sub-steps:

- Data cleaning
  - Drop duplicate rows
  - Coerce dates to a single format
- Modeling
  - Fit the baseline
  - Compare against the tuned model

Task lists track what is done and what is left:

- [x] Draft the outline
- [x] Write the first section
- [ ] Add the figures
- [ ] Final proofread

## Code blocks

Fenced code blocks are syntax highlighted for both light and dark themes.

```python
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        if arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1
```

## Callouts

Callouts pull a note out of the flow. There are six types, each with its own color and icon.

<Callout type="info" title="Info">
  Neutral context or a side note the reader might want.
</Callout>

<Callout type="note">
  Like info, but with a lighter framing for a quick aside.
</Callout>

<Callout type="tip" title="Tip">
  A shortcut or better way to do something. Press `Ctrl + K` for the command palette.
</Callout>

<Callout type="success" title="Done">
  Confirmation that a step worked and it is safe to move on.
</Callout>

<Callout type="warning" title="Careful">
  Something that can bite you if you are not paying attention.
</Callout>

<Callout type="danger" title="Do not">
  A real hazard. Never commit API keys or passwords to a public repository.
</Callout>

## Math with KaTeX

Inline math sits in a single line of prose: the mass-energy relation $E = mc^2$, or the sample mean $\bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i$.

Display math gets its own centered block:

$$
f'(x) = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h}
$$

The Gaussian integral, a favorite:

$$
\int_{-\infty}^{\infty} e^{-x^2}\, dx = \sqrt{\pi}
$$

Matrices and the OLS estimator, the kind of thing a statistics post needs:

$$
\hat{\beta} = (X^\top X)^{-1} X^\top y
\qquad
\begin{bmatrix} a & b \\ c & d \end{bmatrix}
\begin{bmatrix} x \\ y \end{bmatrix}
=
\begin{bmatrix} ax + by \\ cx + dy \end{bmatrix}
$$

## Concept checks

A concept check hides its answer until the reader clicks, which is handy for tutorials.

<ConceptCheck question="What is the time complexity of binary search?">
  **O(log n).** Each comparison halves the search space, so a million elements need at most about twenty steps<Cite id="clrs" num={1} />.
</ConceptCheck>

<ConceptCheck>
  <span slot="question">In $ax^2 + bx + c = 0$, what is the discriminant?</span>
  It is $\Delta = b^2 - 4ac$. It decides the roots: $\Delta > 0$ gives two real roots, $\Delta = 0$ gives one, and $\Delta < 0$ gives a complex conjugate pair.
</ConceptCheck>
